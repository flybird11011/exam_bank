# 错题回顾页设计

## 目标

在现有练习能力之上，新增一个面向“错题回顾”的页面。它以“列表 + 详情”的方式帮助用户复盘已做错、未掌握、错题次数较高的题目。

## 范围

本次只做错题回顾页，不改变导入、审核和现有题库检索主流程。

## 页面结构

### 左侧

- 错题筛选
- 错题列表

列表项至少显示：

- 题号
- 题型
- 错题次数
- 掌握状态

### 中间

- 当前题目题干
- 当前题目的掌握状态
- 当前题目的错题次数
- 操作按钮

### 右侧

- 当前题目的最近作答记录
- 继续下一题
- 标记已掌握

## 默认交互

- 默认只显示题干
- 答案和解析默认折叠
- 用户点击“显示答案”后才展开答案和解析
- 切换到下一题时，下一题重新回到折叠状态

## 操作

错题回顾页支持以下操作：

- `显示答案`
- `标记已掌握`
- `答对`
- `答错`
- `跳过`
- `下一题`

其中：

- `跳过` 视为已掌握
- `答对` 视为已掌握
- `答错` 会让 `wrong_count + 1`，并把 `mastered` 置为 `false`

## 数据来源

页面只消费现有练习后端提供的数据：

- `GET /api/practice/questions`
- `GET /api/practice/questions/{question_id}`
- `POST /api/practice/attempts`

题干、答案、解析的富文本块沿用审核页已有的块渲染方式，优先使用后端返回的：

- `stem_blocks`
- `analysis_blocks`
- `option_blocks`

如果块数据不存在，则回退到文本展示。

## 交互约束

- 错题列表默认按“未掌握 + 错题次数高”优先显示
- 详情区默认不自动展开答案和解析
- 页面不强制要求自动切题，但应保留“下一题”入口，方便后续升级为连续回顾模式

## 验收标准

- 能筛出错题和未掌握题
- 能查看题干而不默认泄露答案
- 能手动展开答案与解析
- 能继续查看下一题
- 能把题目标记为已掌握
